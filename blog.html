Discuss the scope of var, let, and const

Before we get into the discuss the scope of var, let, and const in Js, let us learn what is scope and type of the scope.

Scope in JavaScript refers to the current context of code, which determines the accessibility of variables to JavaScript. The two types of scope are local and global: Global variables are those declared outside of any block or function scope. This means we can be accessed the variables anywhere in our code. Local variables are those declared inside of a block or function. We only accessed the variables inside the function or block scope

Now we will discuss the Functional scope and Block scope

Function scope

Variable that is declared inside the function & can't be accessed outside of that function. var keyword maintain the Functional scope,But global scope when it is declared outside the function. Var is the old way to declare a variable

Block scope

Variable that is declared inside a specific block (a pair of curly bracket) & can't be accessed outside of that block. let and const keyword maintain block scope.It's explore ES6 or ES 2015 version.

Let's now we will discuss the var, let, and const keyword and understand with example

The var variable has Functional scope, meaning var variable will be accessible only inside the function it's declared if we try of access outside of the scope, it will show a Reference Error but it's declared the outside of the function then we access the whole program.
Example for var:
function func1() {
var a=10;
}
console.log(a)

  • When we will run the code, we will see an error as a is not defined because var variables are only accessible in the function scope.

    • The let and const variable has block scope, meaning let and const variable will be accessible only inside the block it's declared if we try of access outside of the scope, it will show a Reference Error.
    Example for let:
    for (let i = 0; i < 3; i++) {
    console.log(i);
    }
    console.log(i);

  • when we will run the code, we will see an error as i is not defined because let variables are only accessible within the block they are declared.
    • Example for const:
    {
    const x = 2;
    console.log(x);
    }
    console.log(x);
  • when we run the code, we will see an error as x is not defined because x was defined under the block which was accessible in this block, but when we try to access x outside that block, we will get an error.

    • Here seeing the table; we will show some difference between var and let and const in javascript:

    Type Scope Redeclare Reassign
    var Functional & Global Yes Yes
    let Block No Yes
    const Block No No
    Tell us the use cases of null and undefined

    Undefined means the variable has been declared, but its value has not been assigned.
    Null means an empty value or a blank value. Undefined and Null are primitive data types.

    They are often used in different ways and have distinct use cases: Here I will discuss some use case for Null and Undefined.

    Undefined use case

    8 ways to get undefined

    • When a variable is declared but not initialized, it will give undefined
    • function with on return
    • If a function called with few arguments but parameter that is not passed will be undefined
    • If a function return has nothing on the right side/value will be return undefined
    • set a value directly to undefined
    • Property that doesn't exist on an object will give you undefined
    • accessing array elements outside of the index range
    • deleting an element inside an array

    Null use case

    Some ways to get null

    • set a value directly to null
    • We can assign null to object properties to remove references to objects
    • You can use null to check if a particular value or object exists or has been assigned.
    • To used for referring absence of any object value
    What do you mean by REST API?

    REST API meaning for Representational State Transfer Application Programming Interface. It is an architectural style for designing networked applications.

    An API (Application Programming Interface) is an interface through which one program or web site talks to another. They are used to share data and services, and they come many different formats and types.

    A RESTful API is one of the many possible ways that programs, servers, and web sites can share data and services. REST describes the general rules for how the data and services are represented through the API so that other programs will be able to correctly request and receive the data and services that an API makes available.
